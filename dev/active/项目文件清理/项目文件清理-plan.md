# 项目文件清理 · Plan

## 大哥摘要

这次任务是给项目"打扫卫生"：把确认没用的临时文件、占位文件删掉，把 dev/active 里早已做完的任务目录归到档案区。做完后你在「Dev Docs」侧栏看到的进行中任务会从 94 个降到只剩真正在做的几个，项目根目录也不再有 `_tp.bat` 这类零碎。**不动任何业务代码、不动你正在进行的微信/AI资讯任务、不碰你的个人文件（docs/求职）**。删除的都是有依据的（逐个查过没有代码引用），删前全部列在下面给你过目。

## 目标

- 删掉确认无用的临时/占位文件（清单见下），验收方式：`git status` 干净（除在途任务外），被删文件无任何代码引用（已 grep 验证）。
- dev/active 里已完成的任务目录归档到 dev/archive，验收方式：「Dev Docs」侧栏只剩在途任务。

## 非目标

- 不清理 node_modules / dist 等构建产物（gitignore 自动管理）。
- 不动在途任务相关文件（radar、微信、agent 团队相关的 modified/untracked 文件）。
- 不重构、不改任何业务代码。

### B. 待大哥拍板：dev/active 的 94 个任务目录怎么归档

绝大多数是 4-6 月间早已做完的任务。两条路（用户感知有差异，所以要问）：

- **路 1：批量搬家** —— 我把已完成的目录直接移到 dev/archive，快（一次搞定 ~88 个），但**不会**触发"归档评审"（就是归档时 AI 自动从任务里提炼经验写进记忆的那个机制）——这些任务的经验大多已经沉淀过了，损失不大。
- **路 2：你在 UI 逐个点归档** —— 每个都会触发归档评审产记忆，但要点 ~88 次。

无论哪条路，这几个**在途任务不动**：AI资讯详情抓原文、微信单飞锁加逃生口、微信接入设置、团队模板借鉴鲁班四条、工作流装配agent团队、权限目录补PowerShell整工具、权限面板完全不问开关（均为最近改动）。

### C. 待大哥拍板：留还是删

- `_design-explorations/`（3 套 UI 方向设计稿 + 截图，约 7 个文件）—— 历史设计探索，界面方向早已定型。留着无害，删了也不影响功能。
- `docs/` 下几份旧规划文档（cli-installer-plan.md、terminal-layout-plan.md 等 6 份）—— 历史规划，默认**保留**不动。
- `docs/求职/` —— 你的个人资料（gitignore 忽略，不会进 git），**我不动**，只提醒它在项目目录里，想挪走自便。
