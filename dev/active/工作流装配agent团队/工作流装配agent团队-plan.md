# 工作流装配agent团队 · Plan

> memory 扫过：auto.md 命中"项目级可选配置目录按'不存在=功能隐藏、坏文件只跳过并记日志'处理"（本任务大脑文件缺失的退化行为遵循此条）、"Dev Docs 任务 context/tasks 必须补齐"；manual.md 无相关条目。ARCHITECTURE.md 不存在（扫过）。

## 大哥摘要

以后你在「设置 → 📐工作流」点应用时，系统会**顺带装一支四人通用开发团队**进这个项目（出现在项目的 `.claude/agents/` 文件夹里）：调研员（team-explorer 摸代码画地图）、实施工（team-implementer 接明确任务动手改代码）、验收员（team-verifier 跑项目自己的检查命令报 PASS/FAIL）、审稿员（team-rules-auditor 交付前查合不合项目规矩），外加一份**团队使用说明书**（装进 `.aimon/docs/`，告诉主 AI 什么活派给谁）。每个助手文件里写死"上岗协议"：**干活前必须先读这个项目的 CLAUDE.md（项目规范）、dev/memory（项目记忆）等文件，读了什么、缺了什么都要在回复里报告**——这就是你说的"有了大脑再干活"，而且记忆更新后助手自动用最新的，不用重装。同时修掉一个现存毛病：现在装工作流会把 7 个 VibeSpace 专属助手（脑子里装的是本仓库的路径和套路）错拷给别的项目，以后不再错拷，旧的错拷会在卸载时安全清理。验收时你去任一项目点一次"应用 / 切换"，再去 `.claude/agents/` 看到四个 team- 开头的文件就对了。

## 目标

- 工作流应用时安装四个通用团队助手 + 一份说明书（大哥确认的 4+1 编成）：
  - `team-explorer`：调研、找文件、梳理项目结构（只读：Read/Glob/Grep）。
  - `team-implementer`：接 tasks 阶段的明确切片动手改代码（Read/Edit/Write/Bash）；纪律段最严——外科式只碰派工指定文件、熔断 2 次停、先读大脑再模仿项目现有写法、派工不明确就退回。
  - `team-verifier`：从项目 README / package.json / Makefile 等找项目自己的检查命令，逐条对 verify 跑并报 PASS/FAIL（Read/Glob/Grep/Bash）；只验不修。
  - `team-rules-auditor`：交付前审 diff 是否遵守项目规则（只读）。
  - `.aimon/docs/team-usage.md` 团队使用说明书：通用版派工指南（什么活派给谁、什么活别派——规划/排障不外包等戒律）。
- 刻意不设：规划师（plan 是主 AI+大哥的事）、排障员（debug 断链）、技术栈专家（二期选配包）。
- 两个助手模板必须包含**硬性"大脑加载协议"段**：
  - 开始任何任务前，先读项目根的 `CLAUDE.md`、`dev/memory/manual.md`、`dev/memory/auto.md`、`dev/ARCHITECTURE.md`、`.aimon/skills/` 相关文件。
  - 回复中必须明确声明成功读取了哪些、哪些不存在或不可读（缺失不报错，退化为通用做法并如实声明）。
  - 全部用项目根相对路径表述，**零占位符、零渲染**——模板对任何技术栈项目通用。
- 支持四条完整路径：
  - **首次安装**：创建缺失的 `team-*` 文件。
  - **重复应用**：结果不变，不重复写入，不覆盖用户自定义文件。
  - **模板升级**：重新应用工作流时，只刷新"由工作流安装且用户未改过"的旧版文件（靠模板头部的版本标记 + 内容指纹——文件内容的校验码，用来判断你有没有手动改过——比对确认）。
  - **卸载**：只移除可确认由工作流安装的文件；同时按 legacy 名单清理历史错拷的 `vibespace-*` 文件（仅当确认未被用户修改时删，拿不准就保留）。
- 新安装清单**移除 7 个 vibespace-* 专属助手**（停止向其他项目错拷），但卸载清单保留旧文件名以便收拾历史安装。
- 工作流 apply/remove 现有日志起止保留，meta 里加上实际安装/移除的团队助手数量。
- 自动验证（仿本仓库 `scripts/*-smoke.mjs` 冒烟脚本套路，配 `pnpm smoke:agent-team`）断言：
  - 生成的助手文件含大脑加载协议段；
  - 不含 `vibespace-` 字面量、VibeSpace 专属路径、调色板、fastify/react/具体函数名等假设；
  - 同名自定义助手不被覆盖、不被卸载；
  - 四条路径（装/重复/升级/卸）全部通过。
- 浏览器可观察验收（大哥手动验，不派自动化）：
  - 任一项目「设置→工作流」点"应用 / 切换"成功后，项目 `.claude/agents/` 出现 `team-explorer.md`、`team-rules-auditor.md`；
  - UI 日志面板看到 apply-workflow 起止配对，成功条目 meta 带助手数量；
  - 卸载后两个文件消失，日志同样有数量。
- "已装 N 个助手"的工作流 tab 状态提示为可选项：只有统计来源可靠（能区分工作流安装的与用户自建的）才做，不可靠就本轮跳过，不展示可能骗人的数字。

## 非目标

- 不装技术栈专属助手（route-author / db-scribe / ui-decorator / smoke-author / browser-tester 等 fastify/react 假设的角色）——后续按真实需求另加。
- 不修复 `.aimon/skills/*.md` 同样被错拷的问题——只在 `dev/issues.md` 记一条遗留，另立任务。
- 不扩展 `refreshAllOutdatedDevDocs`（批量刷新入口）——模板升级靠重新应用工作流触发，控制改动面。
- 不改数据库、不改项目业务数据、不改工作流页面的主要操作流程。
- 本仓库自己的 7 个 `vibespace-*` 助手原件不动；本仓库点应用时 team-* 与 vibespace-* 按文件名自然共存，不加特殊分支。

## 实施步骤

1. 盘点装配链路边界（getTemplateFiles / applyHarnessTemplate / uninstallHarnessTemplate / renamed 探测 / 日志字段），把 `.aimon/skills` 错拷问题追加进 `dev/issues.md`。→ 验证：context.md 列出全部受影响符号与行号；issues.md 多一条。
2. 写两个通用模板（放 `templates/agent-team/`）：角色纪律骨架 + 硬性大脑加载协议 + 读取声明要求 + 头部版本标记/内容指纹注释。→ 验证：内容检查无 `vibespace-`、无专属路径/调色板/函数名/技术栈假设，协议段存在。
3. 改安装逻辑：新清单 = 2 个 team-* 模板（来自 templates/agent-team/，不再来自仓库根 .claude/agents/）；存在不覆盖；旧版且未被改过才刷新；维持纯文件拷贝。→ 验证：冒烟脚本覆盖首次/重复/升级/自定义保留四况。
4. 改卸载逻辑：删本次体系安装的 team-* + 按 legacy 名单清理未被改过的旧 vibespace-* 错拷；拿不准保留。→ 验证：冒烟覆盖卸载/旧件清理/自定义保留；改完全量 grep 旧清单引用无残留。
5. 写 `scripts/agent-team-smoke.mjs`（临时目录造假项目，跑四条路径 + 内容安全断言），挂 `pnpm smoke:agent-team`。→ 验证：脚本通过；`pnpm -F @aimon/server build` 通过。
6. apply/remove 日志 meta 加助手数量；状态提示可靠才做。→ 验证：冒烟断言数量字段准确；UI 日志面板人工核对起止配对。
7. 真机验收：在一个非 node 技术栈项目（如 galgame 或 godot_ai）走完 装→重复→模拟升级→卸 四条路径，确认自定义文件无恙。→ 验证：四条路径磁盘结果符合预期；大哥按"大哥摘要"指引点一遍。

## 边界情况

- 项目缺 `CLAUDE.md` / `dev/memory/*` / `dev/ARCHITECTURE.md` / `.aimon/skills/`：助手声明缺失项，继续用可读到的资料干活。
- 项目已有同名 `team-*` 自定义助手：装/重复应用均不覆盖。
- 工作流装的助手被用户改过：即使版本落后也不自动刷新、不卸载。
- 旧 `vibespace-*` 文件无法确认是否原样：宁可保留，不冒险删。
- 文件只读 / 拷贝中断：沿用现有错误处理，日志记失败终点。
- 助手文件部分缺失：重复应用只补缺失的，不动还在的。
- 数量统计只算"仍存在且确认由工作流安装"的 team-* 文件。

## 风险与注意

- 假设 markdown 助手文件头部注释（版本标记/指纹）不影响 agent 执行——实施第 2 步先验证一次。
- "用户未修改"判定必须靠版本标记 + 内容比对，不能只看文件名。
- legacy vibespace-* 清理依赖现有 renamed 探测；探测不可靠就保守跳过删除并写日志说明。
- 安装清单移除 7 个旧助手后，必须确认卸载逻辑仍保留历史清理能力，避免 11 个已装项目收拾不掉。
- 大脑加载协议依赖 agent 遵守提示词，不能 100% 保证——所以验收看的是"回复中声明了读取清单"这个可观察行为。
- 日志 meta 保持可序列化且精简（只记数量与标识，不塞文件内容）。
