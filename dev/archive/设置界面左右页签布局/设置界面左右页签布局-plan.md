# 设置界面左右页签布局 · Plan

## 大哥摘要

现在点「设置」弹出的是一长条往下滚的窗口，五块内容（图片保留天数、会话冬眠、终端快捷键、桌面通知、飞书机器人）全挤在一列里，要找哪块得拖滚动条。

这次把它改成**左右两栏**：左边一竖列是分类按钮（页签），右边只显示你点中那一类的内容。点左边「飞书机器人」，右边就只剩飞书那一坨；点「终端」，右边换成冬眠+快捷键。**功能一个不删、配置一个不少，只是换了个摆法**，更好找。窗口会从窄条变宽一点。

你现有的任何设置值都不会动，保存按钮还在原来的位置照常用。验收就是：打开设置，左边能看到几个分类，点一下右边内容跟着换。

## 目标

把单文件 `SettingsDialog.tsx` 的竖排滚动布局，改成左侧页签 + 右侧内容的左右框架，**不增删任何配置项、不改任何保存/读取逻辑**。

可验证的验收标准：
1. `pnpm -F @aimon/web build` 通过（类型检查 + 构建）。
2. 浏览器里打开「设置」：左侧出现一列分类页签，右侧显示当前页签内容；点不同页签右侧内容切换。
3. 五块原有内容全部还在、值不丢：在某页签改了值、切到别的页签再切回来，改的值仍在（状态不因切页签丢失）。
4. 底部「取消 / 保存」按钮、飞书区自己的「测试连接 / 保存飞书配置」按钮行为不变，保存仍写入对应配置。

## 页签分组（这是唯一需要你拍板的点）

按功能把 5 块归成 3 个页签：

- **通用**：粘贴图片保留天数、桌面通知
- **终端**：会话冬眠、终端快捷键
- **飞书机器人**：飞书全部配置

如果你想要别的分法（比如「通知」单独一个页签、或四个页签更细），在确认时直接说。

## 非目标

- 不改任何配置的读取/保存接口、数据结构、字段。
- 不动键盘录制、飞书测试连接等任何业务逻辑。
- 不给设置项加新功能、不调整任何默认值。

## 实施步骤

1. 在组件里加一个本地状态 `activeTab`（默认「通用」），把现有 5 个 `<section>` 按分组拆进 3 个页签的渲染分支。→ 验证：build 过，三个页签都能渲染。
2. 把弹窗外层从单列 `w-[460px]` 改成左右 flex 容器（左侧页签列固定宽、右侧内容区可滚动），加宽到约 `720px`。→ 验证：浏览器里左右两栏成形，右侧超长时只右侧滚动。
3. 左侧页签按钮：选中态高亮，点击切换 `activeTab`。→ 验证：点击切换右侧内容。
4. 底部「取消/保存」「飞书保存」按钮位置与逻辑保持不变。→ 验证：保存后值落库、关窗。

## 边界情况

- 切页签时**不能丢未保存的输入**：所有字段状态在组件顶层用 `useState`，切页签只是条件渲染，不卸载状态——天然不丢，需在实现时确保不把 state 下沉进子分支。
- 飞书区的「测试结果 / 错误提示」只在飞书页签显示，切走再切回应仍在（同样靠顶层 state）。
- 键盘录制中（recording）切页签：录制监听是全局 capture，切页签不影响；但录制 UI 在终端页签，切走会看不到提示——可接受（切回即见），不特殊处理。
- 窄屏（max-w-[90vw]）：720px 在小屏要能缩，保留 `max-w-[90vw]`，左侧页签列在极窄屏不做响应式折叠（非目标）。

## 风险与注意

- 纯前端单文件改动，无数据/路由/DB 风险。
- 唯一回归风险是「拆 section 时漏搬某块 / state 绑定错位」，靠验收标准 3（切页签值不丢）和逐块对照原文件覆盖。
- 操作日志：本任务是纯布局重排 + 本地 UI 状态（切页签），不新增 mutation、不改现有 save 行为 → 按豁免项处理，不加新日志（现有 `logAction` 调用原样保留）。
