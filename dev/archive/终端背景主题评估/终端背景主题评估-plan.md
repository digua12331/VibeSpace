# 终端背景主题评估

## 大哥摘要

这次要确认两件事：这个项目里的终端黑底能不能改，以及要改成“全部终端统一一个新颜色”，还是“每个终端自己单独一个颜色”。
你之后能直接在浏览器里的终端区域看到结果：打开任意一个终端（页面里跑 Claude / Codex / shell 的那块黑底区域），背景色会变，文字和光标也要保持清楚能看。
这次只动前端主题（浏览器里看到的样式），不碰你的项目数据、不改终端里的命令内容，也不影响已有会话记录。
我已经看过现状，当前黑底不是系统自动给的，而是代码里写死了两处，所以技术上能改；真正要定的是改动范围。

## 目标

- 明确当前终端背景色的来源，并确认是否支持改成非黑色背景。
- 给出两条用户可见方案的取舍，并推荐默认方案：
  1. 统一主题：所有终端一起换成新的背景色。
  2. 独立主题：每个终端可以分别有不同背景色。
- 若确认进入实现，验收标准至少包括：
  - 在浏览器打开任意终端时，背景不再是当前黑色，而是指定新颜色。
  - 文字、光标、选中高亮在新背景上仍清晰可见，没有“看不清命令”的问题。
  - 新开终端和已存在终端行为一致，不出现一部分换色、一部分还是旧色的情况。

## 非目标 (Non-Goals)

- 本轮不改终端功能行为，例如复制粘贴、快捷键、会话管理。
- 本轮不做整站主题系统重构（例如把整个应用都改成浅色或多主题切换器）。
- 在你没有确认前，不直接把“每个终端都能单独自定义颜色”的完整设置面板做出来。

## 实施步骤

1. 梳理现状：确认终端背景色是由组件内 xterm 主题和全局 CSS（样式表）共同控制；验证方式：定位到实际代码来源并列出影响文件。
2. 评估方案 A（统一换色）：把终端背景、前景字色、光标色抽成一组统一主题值；验证方式：浏览器里打开多个终端，全部显示同一新底色。
3. 评估方案 B（每终端独立换色）：给会话（终端实例）增加单独主题配置入口和状态保存；验证方式：两个终端可显示不同底色，刷新后表现符合预期。
4. 比较工作量和风险，给出推荐；验证方式：明确哪条路径改动更小、回归面更窄、验收更直接。

## 边界情况

- 如果只改组件内的 xterm 主题、不改全局 CSS，可能仍会被全局样式压回深色，看起来像“没改成功”。
- 如果背景改浅了，现有白字和浅蓝光标可能对比度不够，终端内容会难看清。
- 如果未来要支持“每个终端不同颜色”，当前会话数据结构里还没有主题字段，需要补存储和恢复逻辑。
- 已打开的终端实例可能需要重新刷新主题或重建实例，否则新颜色只对新开的终端生效。
