# 本地实现 · 切分支改代码

产品自闭环第 2 步：为选定的 issue `<N>` 切一个分支，改代码，本地测试通过。数据契约见 `产品自闭环-总纲`。

## 前置依赖

- `git`，且当前在目标项目的 git 仓库里。

## 步骤

1. 切分支（分支名固定，见总纲数据契约）：

   ```
   git checkout -b feat-ai-issue-<N>
   ```

2. 读懂 issue 要求，改对应的源码文件。遵循该项目自己的代码风格和约定 —— 先看它的 `CLAUDE.md` / `README` / 既有代码。只碰必须碰的，不顺手重构。

3. 跑该项目自己的测试 / 类型检查命令（在 `package.json` scripts、`Makefile`、`justfile`、构建配置里找；静态类型语言必须过类型检查）。测试不过就继续修，不要带着失败的测试往下走。

4. 提交，但**先不要 push**：

   ```
   git add -A
   git commit -m "feat: resolve issue #<N> — <一句话简述>"
   ```

   push 和提 PR 是第 4 步「发布」的事，且需要人工确认 —— 这里只在本地提交。

## 失败处理

- 改不动 / 测试卡住 / 同一处连续失败 2-3 次 → 停下来，把错误、试过的方案、当前疑惑告诉大哥，不要越改越大。
